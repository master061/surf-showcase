# SURF 学生科研展示小程序 — 产品需求文档

## 1. 产品概述

### 1.1 产品背景
高校学生参与科研项目后缺乏有效的展示和宣传渠道。现有的学术交流平台多面向教师和研究人员，缺少一个专属于本科生的科研成果展示空间。

### 1.2 产品定位
面向本校学生的科研项目宣传与展览平台。学生通过身份认证后，可以发布自己的科研项目，向全校师生展示研究成果，促进跨学科交流与合作。

### 1.3 产品目标
- 为学生提供便捷的科研成果展示渠道
- 降低科研项目的宣传门槛，帮助新项目招募团队成员
- 促进不同专业学生之间的学术交流
- 营造校园科研氛围

## 2. 技术架构

| 层级 | 技术 |
|------|------|
| 前端 | 微信原生小程序（WXML + WXSS + JS） |
| 后端 | 微信云开发（云函数 + 云数据库 + 云存储） |
| 认证 | 微信一键登录 + 邮箱密码注册 |
| 存储 | 云数据库 4 个集合 + 云存储图片 |

### 2.1 数据库集合

| 集合 | 说明 |
|------|------|
| `users` | 用户信息（含 openid、角色、学号、院系等） |
| `projects` | 项目信息（含状态、类型、指导老师、成员等） |
| `votes` | 投票记录 |
| `notifications` | 消息通知 |

## 3. 用户角色

### 3.1 游客
- 浏览所有项目、搜索筛选、查看详情
- 不能发布、编辑、投票

### 3.2 认证学生
- 游客所有权限 + 发布/管理项目 + 投票 + 编辑资料
- 项目分为**已完成**和**招人中**两种状态

### 3.3 管理员
- 学生所有权限 + 管理所有项目 + 审核流程 + 用户管理
- 有独立管理后台

## 4. 整体页面结构

```
学生暑期项目展示平台
├── 前台
│   ├── 首页（热门推荐、正在招人、研究领域、项目卡片）
│   ├── 项目列表（搜索、筛选、排序、分页）
│   └── 项目详情页
├── 用户中心
│   ├── 微信一键登录 / 邮箱注册
│   ├── 首次信息补全
│   ├── 个人中心（我的项目、统计）
│   ├── 个人资料编辑
│   └── 消息通知
└── 管理后台
    ├── 后台首页（数据统计概览）
    ├── 内容审核
    └── 用户管理
```

## 5. 功能需求

### 5.1 用户系统

#### 5.1.1 登录
- 微信一键登录（获取微信昵称+头像，首次需补全学生信息）
- 邮箱密码登录 + 注册（传统方式备选）

#### 5.1.2 个人资料
- 查看和编辑：姓名、学号、学校/院系、入学年份、个人简介
- 头像上传：点击更换，从相册选择或拍照，上传至云存储

### 5.2 项目发布与管理

#### 5.2.1 项目状态
- **✅ 已完成**：分享已完成的科研项目经历
- **🔍 招人中**：招募团队成员加入
- 项目类型：**个人项目**、**团队项目**（已去除"班级项目"）

#### 5.2.2 3步创建向导

**Step 1：项目内容**
| 字段 | 必填 | 说明 |
|------|:----:|------|
| 项目名称 | ✅ | |
| 研究领域 | ✅ | 8个预设领域 |
| 项目类型 | ✅ | 个人项目 / 团队项目 |
| 项目年份 | ✅ | |
| 姓名 | ✅ | 自动填充登录用户 |
| 学校/院系 | ✅ | |
| 指导老师 | ❌ | 所有项目通用 |
| 项目成员 | ❌ | 仅团队项目时显示 |
| 项目简介 | ✅ | |
| 研究成果 | ❌ | 整合的方法/过程/结论等 |
| 标签 | ❌ | 逗号分隔 |
| 封面图 | ❌ | 从相册上传 |

**Step 2：联系方式 + 状态**
| 字段 | 必填 | 说明 |
|------|:----:|------|
| 联系方式 | ✅ | 邮箱或微信 |
| 项目地址 | ❌ | GitHub 等 |
| 已完成 | | 填写成果链接 |
| 招人中 | | 填写招募人数、周期、要求 |

**Step 3：预览 + 提交审核**
- 预览完整内容
- 提交后进入审核流程

#### 5.2.3 发布流程
```
创建项目（草稿 DRAFT）→ 提交审核（PENDING）
                         ↓
                管理员 → 通过（APPROVED）→ 正式展示
                       → 驳回（REJECTED）→ 查看原因重新修改
```

### 5.3 项目展示

#### 5.3.1 首页
- Hero 区：SURF 品牌标题 + 英文副标题 + 装饰分隔线
- 统计卡片：项目总数、正在招人、研究领域
- **正在招人**专区（金色边框卡片）
- **研究领域** 8 宫格快捷入口
- **热门推荐**（按获票数排序）
- **最新发布**
- 铃铛图标：未读通知红色角标

#### 5.3.2 项目列表
- 搜索框 + 自动补全建议
- 状态筛选（全部/已完成/招人中）
- 领域/类型/年份筛选
- 排序（最新/最热）
- 分页

#### 5.3.3 项目详情
- 状态标签（已完成绿色 / 招人中金色+脉冲动画）
- 作者信息卡片（头像、姓名、学校、票数）
- 项目简介、研究成果、指导老师
- 招人项目：招募信息区、联系方式、联系按钮
- 已完成项目：参与成员、成果链接
- 标签
- 投票按钮
- 所有者/管理员：编辑、删除按钮

### 5.4 搜索与筛选
- 全局模糊搜索（标题、标签、领域、作者）
- 自动补全建议（标题、标签、领域、作者）
- 多维度筛选：领域 + 类型 + 年份 + 状态

### 5.5 投票系统
- 认证学生可投票/取消投票
- 每人每项目限投一票
- 投票数反映热度

### 5.6 消息通知

**触发场景：**
| 操作 | 通知内容 |
|------|----------|
| 提交审核 | 📤 你的项目已提交，请等待管理员审核 |
| 审核通过 | ✅ 项目已通过审核，正式发布 |
| 审核驳回 | ❌ 项目未通过审核，原因：... |
| 收到投票 | 👍 xxx 赞了你的项目 |

**入口：** 首页铃铛图标（红点角标）→ 通知列表页
- 点击通知标记已读并跳转对应项目
- 一键全部已读

### 5.7 个人中心

#### 我的项目（按状态分组）
- 招人中 / 已完成 分组展示
- 项目数量、总获票数统计
- 编辑 / 查看入口

#### 个人资料编辑
- 头像上传、姓名、学号、院系、年份、简介、邮箱

#### 管理员入口
- 管理后台卡片（仅管理员可见）

### 5.8 管理后台

#### 后台首页
- 数据统计：待审核数、项目总数、用户总数、总投票数
- 快捷操作入口

#### 内容审核
- 待审核项目列表（显示标题、作者、摘要）
- ✅ 通过 / ❌ 驳回（可填写驳回原因）
- 已通过项目列表

#### 用户管理
- 首次使用：成为管理员按钮（无管理员时显示）
- 用户列表（姓名、邮箱、角色）
- 设为管理员 / 取消管理员

### 5.9 图片上传
- 相册选择或拍照
- 上传至云存储
- 支持：头像、封面图

## 7. 非功能需求

### 7.1 用户体验
- 骨架屏加载状态
- 已完成/招人项目视觉区分（颜色、边框动画）
- 操作反馈及时
- 空状态提示

### 7.2 权限安全
- 未登录只能浏览
- 学生只能管理自己的项目
- 管理员可管理所有项目 + 审核
- 密码 bcrypt 加密

### 7.3 数据
- 云数据库持久化
- 搜索用正则匹配
- 分页加载

## 8. UI 设计规范

### 8.1 设计理念
整体采用**"文艺极简 + 卡片化社交"**风格，参考读书/社交类 App 的 UI 语言。核心特征：暖白色背景、大圆角卡片、药丸形按钮/标签、衬线字体大标题、橄榄绿主色调。

### 8.2 色彩系统

| 用途 | 色值 | 说明 |
|------|------|------|
| 页面背景 | `#F7F6F2` | 暖白色/奶白色，全局底色 |
| 卡片背景 | `#FFFFFF` | 纯白，叠在暖白背景上自然分层 |
| 主按钮/强调 | `#5A6B3A` | 深橄榄绿/苔藓绿，用于主行动按钮 |
| 主文字 | `#1A1A1A` | 近黑 |
| 次要文字 | `#6B6B6B` / `#8A8A8A` | 中灰、浅灰 |
| 蓝色强调 | `#2AAECF` | 湖蓝色，用于领域标签等辅助信息 |
| 金色强调 | `#B8860B` | 用于"招人中"状态 |
| 浅灰背景 | `#EFEEE9` | 搜索框、二级按钮、公告栏背景 |

### 8.3 字体规范
- **大标题**（页面标题、Hero 标题）：`Georgia, 'Times New Roman', 'Songti SC', serif`，`font-weight: 600`，`letter-spacing: -0.02em`
- **正文/标签**：系统无衬线 `-apple-system, 'PingFang SC', sans-serif`
- **标题字号**：48rpx（页面标题）、40rpx（卡片标题）
- **正文字号**：28rpx ~ 30rpx
- **辅助字号**：20rpx ~ 24rpx

### 8.4 圆角与阴影
- **卡片圆角**：`28rpx` ~ `32rpx`
- **按钮圆角**：完全药丸形 `border-radius: 999px`
- **图片圆角**：卡片内图片顶满时，卡片圆角自动裁剪；独立图片 `20rpx` ~ `28rpx`
- **Tab Bar**：顶部圆角 `24px 24px 0 0`
- **阴影**：极淡阴影 `box-shadow: 0 2rpx 12rpx rgba(0,0,0,0.04)`，不靠重阴影分层

### 8.5 按钮与徽章

**药丸按钮（Pill Button）**
- `border-radius: 999px`
- 主按钮：`background: #5A6B3A; color: #fff`
- 描边按钮：`border: 1.5px solid #1A1A1A; background: transparent`
- 幽灵按钮：`background: #EFEEE9; color: #1A1A1A`

**徽章（Badge）**
- `border-radius: 999px`
- `padding: 6rpx 20rpx`
- 已完成：`background: #E8EDDE; color: #5A6B3A`
- 招人中：`background: #FFF8E1; color: #B8860B`
- 领域：`background: #E6F4F7; color: #2AAECF`

### 8.6 底部导航（Tab Bar）
- **自定义 Tab Bar**：`"custom": true`
- 5 个 Tab：首页 / 项目 / 探索 / 收藏 / 我的
- 背景：白色，顶部圆角 `24px 24px 0 0`
- 图标：1.5px 线框 SVG，未选中 `#8A8A8A`，选中 `#1A1A1A`
- 文字：未选中 `#8A8A8A`，选中 `#1A1A1A` + `font-weight: 600`

### 8.7 项目卡片组件（project-card）

**布局要求**：
- 整体白色大圆角卡片 `border-radius: 32rpx`
- 如果有封面图，图片顶满卡片上半部分 `height: 280rpx`
- 内容区 `padding: 24rpx`

**内容规范**：
- 顶部：**药丸状态标签**（已完成/招人中）+ **领域标签** + 年份标签，横向排列，**垂直居中对齐**
- **不显示项目简介/摘要小字**
- 标题：`font-size: 30rpx; font-weight: 700; color: #1A1A1A`
- 底部：作者头像（首字母）+ 作者名称/学校 + 右侧投票 pill（`❤️ 数字`）
- 底部与内容区间用 `1rpx solid #F2F2F0` 分隔线

### 8.8 各页面布局要求

| 页面 | 核心特征 |
|------|----------|
| **首页** | 白色圆角 Hero 卡片（SURF 衬线标题 + 统计数字 + 药丸按钮）+ 研究领域 4 列网格 + 项目卡片列表 |
| **项目列表（Clubs）** | 顶部衬线大标题 "项目" + 药丸搜索框 + Tab 切换（全部/招人）+ 横向药丸领域标签 + 项目卡片列表 + 圆角分页 |
| **探索** | 衬线标题 + 搜索框 + 横向标签（Featured/领域）+ 热门项目大卡片 |
| **收藏** | 衬线标题 + 项目卡片列表 + 空状态（🔖 图标 + 提示文字） |
| **项目详情** | 全宽封面图 + 左上角返回按钮 + 悬浮白色大圆角内容卡片（`margin-top: -60rpx` 顶入封面）+ 药丸标签 + 衬线标题 + 底部固定操作栏（投票/联系按钮） |

### 8.9 导航栏
- 全局导航栏背景：`#F7F6F2`，文字黑色
- 详情页使用自定义导航栏 `navigationStyle: custom`，自行绘制返回按钮（左上角圆形半透明按钮）

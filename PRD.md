# SURF 学生科研展示小程序 — 产品需求文档

## 1. 产品概述

### 1.1 产品背景
高校学生参与科研项目后缺乏有效的展示和宣传渠道。现有的学术交流平台多面向教师和研究人员，缺少一个专属于本科生的科研成果展示空间。

### 1.2 产品定位
面向本校学生的科研项目宣传与展览平台。学生通过身份认证后，可以发布自己的科研项目，向全校师生展示研究成果，促进跨学科交流与合作。

### 1.3 产品目标
- 为学生提供便捷的科研成果展示渠道
- 降低科研项目的宣传门槛，帮助新项目招募团队成员
- 促进不同专业学生之间的学术交流
- 营造校园科研氛围

## 2. 用户角色

### 2.1 游客
- 可以浏览平台上的所有项目
- 可以搜索和筛选项目
- 可以查看项目详情
- 不能发布、编辑项目
- 不能投票

### 2.2 认证学生
- 拥有游客的所有权限
- 可以发布自己的科研项目（已完成或招人中）
- 可以编辑和管理自己的项目
- 可以为感兴趣的项目投票
- 可以编辑个人资料

### 2.3 管理员
- 拥有认证学生的所有权限
- 可以管理所有项目（编辑/删除/审核）
- 可以管理用户
- 平台运营管理

## 3. 整体页面结构

```
学生暑期项目展示平台
├── 前台
│   ├── 首页（项目列表页）
│   └── 项目详情页
├── 用户中心
│   ├── 注册 / 登录页
│   ├── 个人中心页
│   └── 项目管理页
│       ├── 项目创建页（3步向导）
│       │   ├── 步骤1：填写项目内容
│       │   ├── 步骤2：填写联系方式
│       │   └── 步骤3：预览提交
│       └── 海报生成器页
└── 管理员后台
    ├── 用户管理页
    ├── 内容审核页
    └── 平台管理页
```

## 4. 功能需求

### 4.1 用户系统

#### 4.1.1 注册
- 学生使用邮箱或学号注册
- 注册时需填写：姓名、学号、学校/院系、入学年份
- 邮箱验证（发送验证邮件）
- 设置账号密码
- 角色权限分配（普通用户 / 管理员）

#### 4.1.2 登录
- 邮箱 / 学号 + 密码登录
- 微信一键登录
- 登录后保持会话状态
- 密码找回功能

#### 4.1.3 个人资料管理
- 查看和编辑：姓名、学号、学校/院系、入学年份、个人简介、头像
- 密码修改
- 个人资料在项目展示中对外可见
- 我的项目列表入口

### 4.2 项目发布与管理

#### 4.2.1 项目状态
项目分为两种状态，发布时需选择：

**已完成项目（COMPLETED）**
- 用于分享已经完成的科研项目经历
- 展示内容包括项目成果、研究方法、实验数据等

**招人项目（RECRUITING）**
- 用于发布刚立项、正在招募团队成员的新项目
- 展示内容包括项目计划、招募要求等
- 项目完成后可标记为"已完成"，转为已完成项目展示

#### 4.2.2 项目创建流程（3步向导）

**步骤1：填写项目内容**

每个字段支持富文本编辑，可选上传图片，上传后支持图片裁剪和大小调整。

| 字段 | 必填 | 说明 |
|------|------|------|
| 项目名称 | ✅ | 项目标题 |
| 摘要 | ✅ | 项目概述，支持富文本+图片 |
| 方法论 | ❌ | 研究方法论描述，支持富文本+图片 |
| 研究方法 | ❌ | 具体研究方法，支持富文本+图片 |
| 过程与结果 | ❌ | 图文混排，支持富文本+图片 |
| 结论 | ❌ | 项目结论，支持富文本+图片 |
| 致谢 | ❌ | 致谢内容，支持富文本+图片 |
| 参考文献 | ❌ | 列表形式，支持上传图片 |
| 项目年份 | ✅ | 所属年份 |
| 项目类型 | ✅ | 个人项目 / 团队项目 / 班级项目 |
| 研究领域 | ✅ | 从预设领域中选择 |
| 封面图 | ❌ | 项目封面 |
| 标签 | ❌ | 便于搜索 |

**步骤2：填写联系方式**

| 字段 | 必填 | 说明 |
|------|------|------|
| 联系方式 | ✅ | 邮箱或微信 |
| 项目地址 | ❌ | GitHub 等项目地址 |
| 社交媒体链接 | ❌ | 个人主页或社交链接 |

**招人项目额外字段：**
- 招募角色/人数
- 预期周期
- 招募要求

**已完成项目额外字段：**
- 参与成员列表
- 指导老师
- 成果链接

**步骤3：预览提交**
- 预览项目完整内容
- 确认无误后提交审核
- 支持返回修改

#### 4.2.3 发布流程

```
保存草稿 → 预览功能 → 提交审核 → 管理员审批 → 正式发布
```

- 支持保存草稿，未完成的项目可暂存
- 提交前可预览完整效果
- 提交后进入审核流程
- 管理员审批通过后正式发布上线
- 审核不通过可查看原因并修改后重新提交

#### 4.2.4 版本控制
- 支持历史版本查看
- 支持版本回滚
- 每次修改自动生成新版本

#### 4.2.5 编辑项目
- 项目所有者可以编辑已发布的项目
- 招人项目完成后可以切换状态为"已完成"
- 管理员可以编辑任何项目

#### 4.2.6 删除项目
- 项目所有者可以删除自己的项目
- 管理员可以删除任何项目

### 4.3 项目展示

#### 4.3.1 首页（项目列表页）
- 热门推荐（按获票数排序，混排两种状态项目）
- 最新发布的项目
- "招人项目"快捷入口（快速查看正在招募的项目）
- 研究领域快捷入口（分类导航）
- 平台统计数据（项目总数、领域数量、招人项目数等）

#### 4.3.2 项目列表
- 项目卡片展示（标题、摘要、领域、年份、状态标签、类型、标签、作者、投票数）
- 已完成和招人项目在卡片上通过状态标签明显区分
- 状态筛选（全部/已完成/招人中）
- 多种排序方式（最新发布、最热投票）
- 分页加载

#### 4.3.3 项目详情页

**已完成项目详情：**
- 封面图片
- 项目标题、领域、年份、类型
- "已完成"状态标识
- 完整内容展示（摘要、方法论、研究方法、过程与结果、结论、致谢、参考文献）
- 参与成员列表、指导老师信息、成果链接
- 标签列表
- 作者信息（姓名、学校）
- 投票功能
- 项目所有者显示编辑入口
- 海报生成入口

**招人项目详情：**
- 封面图片
- 项目标题、领域、年份、类型
- "招人中"醒目状态标识
- 招募信息区：招募角色/人数、联系方式、预期周期
- 项目计划详细介绍
- 标签列表
- 作者信息（姓名、学校）
- 投递意向表达（感兴趣按钮/联系）
- 投票功能
- 项目所有者显示编辑入口

### 4.4 搜索与筛选

#### 4.4.1 全局搜索
- 按项目标题、标签、研究方向等进行模糊搜索
- 搜索时提供自动补全建议

#### 4.4.2 多维度筛选
- 按研究领域筛选（计算机科学、人工智能、生物医药、物理数学、化学材料、工程技术、社会科学、人文艺术等）
- 按项目类型筛选（个人项目、团队项目、班级项目）
- 按项目状态筛选（全部、已完成、招人中）
- 按年份筛选
- 按标签筛选
- 支持多维度组合筛选

#### 4.4.3 搜索建议
- 输入关键词时自动弹出相关建议
- 建议分类：相关标题、相关标签、相关领域、相关作者
- 点击建议快速跳转搜索结果

### 4.5 投票系统
- 认证学生可以对感兴趣的项目投票（点赞）
- 可以取消投票
- 投票数反映项目的受欢迎程度
- 每人每项目限投一票

### 4.6 海报生成器
- 将项目内容自动生成为海报图片
- 支持多种海报模板选择
- 海报包含：项目标题、摘要、作者信息、领域标签
- 生成后可保存到相册或分享

### 4.7 个人中心

#### 4.7.1 我的项目
- 查看自己发布的所有项目
- 项目按状态分组展示（招人中/已完成/草稿）
- 项目数量统计
- 总获票数统计
- 招人项目可快速切换为已完成

#### 4.7.2 个人资料入口
- 编辑个人资料
- 修改密码

### 4.8 管理员后台

#### 4.8.1 用户管理页
- 查看所有用户列表
- 管理用户角色（设为管理员/普通用户）
- 禁用/启用用户账号

#### 4.8.2 内容审核页
- 查看待审核项目列表
- 审核通过 / 驳回（填写驳回原因）
- 已通过项目列表
- 已驳回项目列表

#### 4.8.3 平台管理页
- 平台数据统计
- 研究领域管理
- 公告发布

## 6. 非功能需求

### 6.1 用户体验
- 界面简洁清晰，便于浏览
- 已完成和招人项目在视觉上有明显区分
- 加载状态要有反馈（骨架屏）
- 操作反馈及时（成功/失败提示）

### 6.2 权限安全
- 未登录用户只能浏览，不能发布
- 学生只能管理自己的项目
- 密码加密存储
- 邮箱验证确保注册真实性

### 6.3 数据
- 项目数据持久化保存
- 搜索响应快速
- 数据按需加载，分页展示

## 7. 迭代规划

### 第一阶段（MVP）
- 用户注册登录（邮箱密码 + 微信一键登录）
- 项目发布与管理（已完成/招人中两种状态，3步创建向导）
- 项目列表与详情展示
- 投票功能
- 个人中心

### 第二阶段
- 搜索与多维筛选
- 自动补全建议
- 个人资料管理
- 富文本编辑 + 图片上传
- 发布流程（草稿 / 预览 / 提交审核）

### 第三阶段
- 管理员后台（用户管理 / 内容审核 / 平台管理）
- 海报生成器
- 版本控制
- 评论/留言功能
- 消息通知
- 密码找回 / 邮箱验证

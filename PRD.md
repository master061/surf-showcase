# SURF 学生科研展示小程序 — 产品需求文档

## 1. 产品概述

### 1.1 产品背景
高校学生参与科研项目后缺乏有效的展示和宣传渠道。现有的学术交流平台多面向教师和研究人员，缺少一个专属于本科生的科研成果展示空间。

### 1.2 产品定位
面向本校学生的科研项目宣传与展览平台。学生通过身份认证后，可以发布自己的科研项目，向全校师生展示研究成果，促进跨学科交流与合作。

### 1.3 产品目标
- 为学生提供便捷的科研成果展示渠道
- 降低科研项目的宣传门槛，帮助新项目招募团队成员
- 促进不同专业学生之间的学术交流
- 营造校园科研氛围

## 2. 用户角色

### 2.1 游客
- 可以浏览平台上的所有项目
- 可以搜索和筛选项目
- 可以查看项目详情
- 不能发布、编辑项目
- 不能投票

### 2.2 认证学生
- 拥有游客的所有权限
- 可以发布自己的科研项目（已完成或招人中）
- 可以编辑和管理自己的项目
- 可以为感兴趣的项目投票
- 可以编辑个人资料

### 2.3 管理员
- 拥有认证学生的所有权限
- 可以管理所有项目（编辑/删除）
- 可以管理用户

## 3. 功能需求

### 3.1 用户系统

#### 3.1.1 注册
- 学生使用学校邮箱或个人邮箱注册
- 注册时需填写：姓名、学号、学校/院系、入学年份
- 设置账号密码
- 注册成功后自动登录

#### 3.1.2 登录
- 使用邮箱和密码登录
- 登录后保持会话状态

#### 3.1.3 个人资料管理
- 查看和编辑个人资料：姓名、学号、学校/院系、入学年份、个人简介、头像
- 个人资料在项目展示中对外可见

### 3.2 项目发布与管理

#### 3.2.1 项目状态
项目分为两种状态，发布时需选择：

**已完成项目（COMPLETED）**
- 用于分享已经完成的科研项目经历
- 展示内容包括项目成果、研究方法、实验数据等
- 填写字段：标题、领域、摘要、详细介绍、年份、类型、封面、标签、参与成员、指导老师、成果链接（可选）

**招人项目（RECRUITING）**
- 用于发布刚立项、正在招募团队成员的新项目
- 展示内容包括项目计划、招募要求等
- 填写字段：标题、领域、摘要、详细介绍、年份、类型、封面、标签、招募角色/人数、联系方式、预期周期
- 项目完成后可标记为"已完成"，转为已完成项目展示

#### 3.2.2 发布项目
认证学生可以发布科研项目，根据项目状态填写对应信息：
- 项目标题
- 研究领域（从预设领域中选择）
- 项目摘要（简要概述）
- 详细介绍（项目内容、方法、成果 或 项目计划、招募要求）
- 项目年份
- 项目类型（个人项目/团队项目/班级项目）
- 封面图片
- 标签（便于搜索）
- 项目状态（已完成/招人中）

已完成项目额外字段：
- 参与成员列表
- 指导老师
- 成果链接（论文、演示视频等）

招人项目额外字段：
- 招募角色/人数
- 联系方式
- 预期周期

#### 3.2.3 编辑项目
- 项目所有者可以编辑已发布的项目
- 招人项目完成后可以切换状态为"已完成"
- 管理员可以编辑任何项目

#### 3.2.4 删除项目
- 项目所有者可以删除自己的项目
- 管理员可以删除任何项目

### 3.3 项目展示

#### 3.3.1 首页
- 热门推荐（按获票数排序，混排两种状态项目）
- 最新发布的项目
- "招人项目"快捷入口（快速查看正在招募的项目）
- 研究领域快捷入口（分类导航）
- 平台统计数据（项目总数、领域数量、招人项目数等）

#### 3.3.2 项目列表页
- 项目卡片展示（标题、摘要、领域、年份、状态标签、类型、标签、作者、投票数）
- 已完成和招人项目在卡片上通过状态标签明显区分
- 状态筛选（全部/已完成/招人中）
- 多种排序方式（最新发布、最热投票）
- 分页加载

#### 3.3.3 项目详情页

**已完成项目详情：**
- 封面图片
- 项目标题、领域、年份、类型
- "已完成"状态标识
- 成果展示区
- 详细描述
- 参与成员列表
- 指导老师信息
- 成果链接
- 标签列表
- 作者信息（姓名、学校）
- 投票功能
- 项目所有者显示编辑入口

**招人项目详情：**
- 封面图片
- 项目标题、领域、年份、类型
- "招人中"醒目状态标识（带招募标签）
- 招募信息区：招募角色/人数、联系方式、预期周期
- 项目计划详细介绍
- 标签列表
- 作者信息（姓名、学校）
- 投递意向表达（感兴趣按钮/联系）
- 投票功能
- 项目所有者显示编辑入口

### 3.4 搜索与筛选

#### 3.4.1 全局搜索
- 按项目标题、标签、研究方向等进行模糊搜索
- 搜索时提供自动补全建议

#### 3.4.2 多维度筛选
- 按研究领域筛选（计算机科学、人工智能、生物医药、物理数学、化学材料、工程技术、社会科学、人文艺术等）
- 按项目类型筛选（个人项目、团队项目、班级项目）
- 按项目状态筛选（全部、已完成、招人中）
- 按年份筛选
- 按标签筛选
- 支持多维度组合筛选

#### 3.4.3 搜索建议
- 输入关键词时自动弹出相关建议
- 建议分类：相关标题、相关标签、相关领域、相关作者
- 点击建议快速跳转搜索结果

### 3.5 投票系统
- 认证学生可以对感兴趣的项目投票（点赞）
- 可以取消投票
- 投票数反映项目的受欢迎程度
- 每人每项目限投一票

### 3.6 个人中心

#### 3.6.1 我的项目
- 查看自己发布的所有项目
- 项目按状态分组展示（招人中/已完成）
- 项目数量统计
- 总获票数统计
- 招人项目可快速切换为已完成

#### 3.6.2 个人资料入口
- 编辑个人资料
- 查看个人信息

## 4. 页面结构

| 页面 | 说明 |
|------|------|
| 首页 | 热门推荐、招人项目入口、最新项目、领域入口 |
| 项目列表 | 搜索、状态/领域/年份/类型筛选、排序、项目卡片列表 |
| 项目详情 | 完整项目信息、状态标识、招募信息或成果展示、投票 |
| 发布项目 | 表单（含状态选择，不同状态显示不同字段） |
| 编辑项目 | 修改已发布的项目（支持状态切换） |
| 登录 | 邮箱密码登录 |
| 注册 | 学生身份注册 |
| 个人中心 | 我的项目（按状态分组）、统计信息 |
| 个人资料 | 查看和编辑个人信息 |

## 5. 非功能需求

### 5.1 用户体验
- 界面简洁清晰，便于浏览
- 已完成和招人项目在视觉上有明显区分
- 加载状态要有反馈（骨架屏）
- 操作反馈及时（成功/失败提示）

### 5.2 权限安全
- 未登录用户只能浏览，不能发布
- 学生只能管理自己的项目
- 密码加密存储

### 5.3 数据
- 项目数据持久化保存
- 搜索响应快速
- 数据按需加载，分页展示

## 6. 迭代规划

### 第一阶段（MVP）
- 用户注册登录
- 项目发布与管理（支持已完成/招人中两种状态）
- 项目列表与详情展示（状态区分展示）
- 投票功能

### 第二阶段
- 搜索与多维筛选（含状态筛选）
- 自动补全建议
- 个人资料管理
- 首页优化（招人项目入口）

### 第三阶段
- 管理员后台
- 评论/留言功能
- 消息通知（招人项目被关注提醒）
- 分享功能

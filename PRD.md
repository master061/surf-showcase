# SURF 学生科研展示小程序 — 产品需求文档

## 1. 产品概述

### 1.1 产品背景
高校学生参与科研项目后缺乏有效的展示和宣传渠道。现有的学术交流平台多面向教师和研究人员，缺少一个专属于本科生的科研成果展示空间。

### 1.2 产品定位
面向本校学生的科研项目宣传与展览平台。学生通过身份认证后，可以发布自己的科研项目，向全校师生展示研究成果，促进跨学科交流与合作。

### 1.3 产品目标
- 为学生提供便捷的科研成果展示渠道
- 降低科研项目的宣传门槛，帮助新项目招募团队成员
- 促进不同专业学生之间的学术交流
- 营造校园科研氛围

## 2. 技术架构

| 层级 | 技术 |
|------|------|
| 前端 | 微信原生小程序（WXML + WXSS + JS） |
| 后端 | 微信云开发（云函数 + 云数据库 + 云存储） |
| 认证 | 微信一键登录 + 邮箱密码注册 |
| 存储 | 云数据库 4 个集合 + 云存储图片 |

### 2.1 数据库集合

| 集合 | 说明 |
|------|------|
| `users` | 用户信息（含 openid、角色、学号、院系等） |
| `projects` | 项目信息（含状态、类型、指导老师、成员等） |
| `votes` | 投票记录 |
| `notifications` | 消息通知 |

## 3. 用户角色

### 3.1 游客
- 浏览所有项目、搜索筛选、查看详情
- 不能发布、编辑、投票

### 3.2 认证学生
- 游客所有权限 + 发布/管理项目 + 投票 + 编辑资料
- 项目分为**已完成**和**招人中**两种状态

### 3.3 管理员
- 学生所有权限 + 管理所有项目 + 审核流程 + 用户管理
- 有独立管理后台

## 4. 整体页面结构

```
学生暑期项目展示平台
├── 前台
│   ├── 首页（热门推荐、正在招人、研究领域、项目卡片）
│   ├── 项目列表（搜索、筛选、排序、分页）
│   └── 项目详情页
├── 用户中心
│   ├── 微信一键登录 / 邮箱注册
│   ├── 首次信息补全
│   ├── 个人中心（我的项目、统计）
│   ├── 个人资料编辑
│   └── 消息通知
└── 管理后台
    ├── 后台首页（数据统计概览）
    ├── 内容审核
    └── 用户管理
```

## 5. 功能需求

### 5.1 用户系统

#### 5.1.1 登录
- 微信一键登录（获取微信昵称+头像，首次需补全学生信息）
- 邮箱密码登录 + 注册（传统方式备选）

#### 5.1.2 个人资料
- 查看和编辑：姓名、学号、学校/院系、入学年份、个人简介
- 头像上传：点击更换，从相册选择或拍照，上传至云存储

### 5.2 项目发布与管理

#### 5.2.1 项目状态
- **✅ 已完成**：分享已完成的科研项目经历
- **🔍 招人中**：招募团队成员加入
- 项目类型：**个人项目**、**团队项目**（已去除"班级项目"）

#### 5.2.2 3步创建向导

**Step 1：项目内容**
| 字段 | 必填 | 说明 |
|------|:----:|------|
| 项目名称 | ✅ | |
| 研究领域 | ✅ | 8个预设领域 |
| 项目类型 | ✅ | 个人项目 / 团队项目 |
| 项目年份 | ✅ | |
| 姓名 | ✅ | 自动填充登录用户 |
| 学校/院系 | ✅ | |
| 指导老师 | ❌ | 所有项目通用 |
| 项目成员 | ❌ | 仅团队项目时显示 |
| 项目简介 | ✅ | |
| 研究成果 | ❌ | 整合的方法/过程/结论等 |
| 标签 | ❌ | 逗号分隔 |
| 封面图 | ❌ | 从相册上传 |

**Step 2：联系方式 + 状态**
| 字段 | 必填 | 说明 |
|------|:----:|------|
| 联系方式 | ✅ | 邮箱或微信 |
| 项目地址 | ❌ | GitHub 等 |
| 已完成 | | 填写成果链接 |
| 招人中 | | 填写招募人数、周期、要求 |

**Step 3：预览 + 提交审核**
- 预览完整内容
- 提交后进入审核流程

#### 5.2.3 发布流程
```
创建项目（草稿 DRAFT）→ 提交审核（PENDING）
                         ↓
                管理员 → 通过（APPROVED）→ 正式展示
                       → 驳回（REJECTED）→ 查看原因重新修改
```

### 5.3 项目展示

#### 5.3.1 首页
- Hero 区：SURF 品牌标题 + 英文副标题 + 装饰分隔线
- 统计卡片：项目总数、正在招人、研究领域
- **正在招人**专区（金色边框卡片）
- **研究领域** 8 宫格快捷入口
- **热门推荐**（按获票数排序）
- **最新发布**
- 铃铛图标：未读通知红色角标

#### 5.3.2 项目列表
- 搜索框 + 自动补全建议
- 状态筛选（全部/已完成/招人中）
- 领域/类型/年份筛选
- 排序（最新/最热）
- 分页

#### 5.3.3 项目详情
- 状态标签（已完成绿色 / 招人中金色+脉冲动画）
- 作者信息卡片（头像、姓名、学校、票数）
- 项目简介、研究成果、指导老师
- 招人项目：招募信息区、联系方式、联系按钮
- 已完成项目：参与成员、成果链接
- 标签
- 投票按钮
- 所有者/管理员：编辑、删除按钮

### 5.4 搜索与筛选
- 全局模糊搜索（标题、标签、领域、作者）
- 自动补全建议（标题、标签、领域、作者）
- 多维度筛选：领域 + 类型 + 年份 + 状态

### 5.5 投票系统
- 认证学生可投票/取消投票
- 每人每项目限投一票
- 投票数反映热度

### 5.6 消息通知

**触发场景：**
| 操作 | 通知内容 |
|------|----------|
| 提交审核 | 📤 你的项目已提交，请等待管理员审核 |
| 审核通过 | ✅ 项目已通过审核，正式发布 |
| 审核驳回 | ❌ 项目未通过审核，原因：... |
| 收到投票 | 👍 xxx 赞了你的项目 |

**入口：** 首页铃铛图标（红点角标）→ 通知列表页
- 点击通知标记已读并跳转对应项目
- 一键全部已读

### 5.7 个人中心

#### 我的项目（按状态分组）
- 招人中 / 已完成 分组展示
- 项目数量、总获票数统计
- 编辑 / 查看入口

#### 个人资料编辑
- 头像上传、姓名、学号、院系、年份、简介、邮箱

#### 管理员入口
- 管理后台卡片（仅管理员可见）

### 5.8 管理后台

#### 后台首页
- 数据统计：待审核数、项目总数、用户总数、总投票数
- 快捷操作入口

#### 内容审核
- 待审核项目列表（显示标题、作者、摘要）
- ✅ 通过 / ❌ 驳回（可填写驳回原因）
- 已通过项目列表

#### 用户管理
- 首次使用：成为管理员按钮（无管理员时显示）
- 用户列表（姓名、邮箱、角色）
- 设为管理员 / 取消管理员

### 5.9 图片上传
- 相册选择或拍照
- 上传至云存储
- 支持：头像、封面图

## 7. 非功能需求

### 7.1 用户体验
- 骨架屏加载状态
- 已完成/招人项目视觉区分（颜色、边框动画）
- 操作反馈及时
- 空状态提示

### 7.2 权限安全
- 未登录只能浏览
- 学生只能管理自己的项目
- 管理员可管理所有项目 + 审核
- 密码 bcrypt 加密

### 7.3 数据
- 云数据库持久化
- 搜索用正则匹配
- 分页加载
